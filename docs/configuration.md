# Configuration

This page details the configuration options available in Gojekyll's `_config.yml` file.

Gojekyll aims to be Jekyll-compatible, so most configuration options from Jekyll are supported. For comprehensive Jekyll configuration documentation, see the [Jekyll docs](https://jekyllrb.com/docs/configuration/).

## Site Structure

### Source and Destination

- **`source`**: Directory containing your site source files (default: current directory)
- **`destination`**: Directory where the built site will be placed (default: `_site`)
- **`layouts_dir`**: Directory for layout templates (default: `_layouts`)
- **`data_dir`**: Directory for data files (default: `_data`)
- **`includes_dir`**: Directory for include files (default: `_includes`)

**Example:**
```yaml
source: ./src
destination: ./public
layouts_dir: _layouts
data_dir: _data
includes_dir: _includes
```

### Collections

Define custom collections of documents. Collections can be configured as a list or a map with options.

**Example:**
```yaml
collections:
  authors:
    output: true
  documentation:
    output: true
    permalink: /docs/:path/
```

### Theme

Specify a Jekyll theme to use for your site.

**Example:**
```yaml
theme: minima
```

## Content Handling

### Include and Exclude

- **`include`**: Files and directories to include even if they match exclude patterns (e.g., `.htaccess`)
- **`exclude`**: Files and directories to exclude from processing
- **`keep_files`**: Files to keep in the destination directory that aren't generated by Gojekyll

**Example:**
```yaml
include: [".htaccess"]
exclude:
  - "Gemfile"
  - "Gemfile.lock"
  - "node_modules"
  - "vendor"
keep_files: [".git"]
```

### Markdown

- **`markdown_ext`**: Markdown file extensions to process (default: `markdown,mkdown,mkdn,mkd,md`)

**Example:**
```yaml
markdown_ext: "markdown,md"
```

## Filtering Content

- **`show_drafts`**: Include posts in the `_drafts` folder (default: `false`)
- **`future`**: Include posts with future dates (default: `false`)
- **`unpublished`**: Include posts marked as `published: false` (default: `false`)

**Example:**
```yaml
show_drafts: true
future: false
unpublished: false
```

## Plugins

Gojekyll supports [some Jekyll plugins](plugins.html). List plugins to enable in your config.

**Example:**
```yaml
plugins:
  - jekyll-feed
  - jekyll-seo-tag
  - jekyll-sitemap
```

## Conversion

### Excerpt Separator

- **`excerpt_separator`**: Separator to use for post excerpts (default: `"\n\n"`)

**Example:**
```yaml
excerpt_separator: "<!--more-->"
```

### Sass/SCSS

Configure Sass processing:

- **`sass.sass_dir`**: Directory for Sass files (default: `_sass`)

**Example:**
```yaml
sass:
  sass_dir: _sass
```

### Incremental Build

- **`incremental`**: Enable incremental build (only rebuild changed files)

**Example:**
```yaml
incremental: true
```

## Serving

- **`host`**: Host to bind to when serving (default: `127.0.0.1`)
- **`port`**: Port to serve on (default: `4000`)
- **`url`**: The absolute URL of your site (used for generating absolute links)
- **`baseurl`**: The subpath of your site, e.g., `/blog`

**Example:**
```yaml
host: 0.0.0.0
port: 8080
url: "https://example.com"
baseurl: "/blog"
```

## Outputting

### Permalinks

- **`permalink`**: Default permalink style for posts. Can be `date`, `pretty`, `ordinal`, `none`, or a custom pattern.

**Example:**
```yaml
permalink: pretty  # Results in /year/month/day/title/
```

See [PERMALINKS.md](PERMALINKS.html) for detailed permalink documentation.

### Front Matter Defaults

Set default values for front matter variables across multiple posts or pages.

**Example:**
```yaml
defaults:
  - scope:
      path: ""
      type: "posts"
    values:
      layout: "post"
      author: "John Doe"
  - scope:
      path: "projects"
      type: "pages"
    values:
      layout: "project"
```

### Timezone

- **`timezone`**: Set the timezone for site generation (e.g., `America/New_York`)

**Example:**
```yaml
timezone: "UTC"
```

### Verbose

- **`verbose`**: Enable verbose output during build

**Example:**
```yaml
verbose: true
```

## Permalink Timezone

Gojekyll allows you to specify a timezone for generating dates in permalinks. This is useful for ensuring that your post URLs are consistent regardless of the server's local timezone.

**Option:** `permalink_timezone`

**Values:**
- A valid IANA Time Zone Database name (e.g., "UTC", "America/New_York", "Europe/Berlin").
- If left unset or empty, Gojekyll will use the server's local timezone. This matches the default behavior of Jekyll.

**Example `_config.yml`:**

```yaml
# _config.yml
title: My Awesome Blog
permalink_timezone: "UTC" # Generates permalink dates in UTC
```

**Behavior:**

- If `permalink_timezone` is set to a valid timezone, all date-based permalink variables (like `:year`, `:month`, `:day`) will be calculated based on that timezone.
- If `permalink_timezone` is invalid (e.g., "Invalid/Timezone"), Gojekyll will log a warning and fall back to using the server's local timezone.
- This setting affects how dates from your posts' front matter are interpreted for URL generation. For instance, a post dated `2023-01-01 02:00:00 +0500` (5 AM in a +0500 timezone) would use January 1st for permalink generation if `permalink_timezone` is "Asia/Kolkata" (UTC+5:30) or similar, but might use December 31st if `permalink_timezone` is "America/Los_Angeles" (UTC-8), depending on the exact date and time.

By default, if you have posts with explicit timezone offsets in their `date` front matter, Gojekyll (like Jekyll) first converts this date to the site's effective timezone (either local or the one specified by `permalink_timezone`) before extracting year, month, and day for the permalink.
